## 一句話

把「信念」從共用 `.section` 殼裡放出來，植入兩個用排版表達的結構件——**梯**與**尺**——證明這套 Mincho 暗場系統做得了「結構教學」，而不只是「沉默留白」。對應〈自己〉測地板（敢空），這頁測天花板（能教）。

---

## 改了什麼（相對 context.html）

骨架、文案、§01–03、§06 對話框**全部沿用 context.html 原樣**。只在兩處植入新 device：

### §04 五層語言 → 一道「梯」
- L1→L5，**往下讀＝往上爬**。
- 攀升走三個頻道：字級↑、明度↑、左軌節點顏色 **灰→紫**↑。
- L1 形容詞（無效）最暗最小、L5 全然在場（being）最大、紫光節點發光。
- 不用框線、不用箭頭、不用比較欄——efficacy 的上升完全靠排版被身體讀到。

### §05 能被指的決策 → 一把「尺」
- 和梯共用節點語彙，但語意相反：**梯找頂、尺找中心**。
- **太大**（失準）：「幫我設計這個頁面。」叛逃出明朝體，渲染成 SaaS 輸入框 + **JS 逐字打字動畫**（捲到畫面內觸發，亮度近全白，游標閃）。
- **太小**（失準）：「按鈕圓角 6px 還是 8px？」+ 兩顆真按鈕（圓角真差 2px）+ 旁註「用眼睛選，不用數字選」。
- **剛好**（校準）：紫光節點 + 浮起的明朝碑文「標題的份量，應該更像碑文，不要像海報。」

## 給 web 宰相的注意事項

1. **i18n：** 文案以 main 分支匯入時的中文為準，請以正式 `context.html` 的最新文案為真實來源覆核。建議比照 **Vessel 法**——整頁多語版本檔、段落對接，不拆 i18n key（這頁有打字動畫的 `data-text` 與旁註，逐 key 拆易碎）。打字字串在 `.typed` 的 `data-text` 屬性，各語言版本記得一起翻。
2. **無障礙（已守 AA）：** 全頁文字對背景 `#0e0b1a` 實測——本文/例句 `ash-1`–`ash-0` ≈ 9:1–15:1；mono 微標籤 `ash-2` ≈ 6.2:1（過 AA 4.5:1）。`ash-3-soft`（3.1:1，不過）已從所有**文字**移除，只留線/點等純裝飾。**併入時請勿把任何文字降回 ash-3-soft。**
3. **沒動共用 `tokens.css` / `shell.css`：** 所有新樣式都在本檔 `<style>` 內。若要推回 design system，梯/尺的 token（節點漸層、grade/rung 階層）可考慮抽進 base.css。
4. **打字動畫：** 純 vanilla JS（IntersectionObserver，threshold 0.35，觀察 `.grade.too-big` 容器而非空的 inline span——後者尺寸為 0 不會觸發，這是踩過的坑）。無依賴。
5. **TOC / dot-nav 保留**（與〈自己〉相反）——因為信念是*教學*頁，導覽幫得上讀者；〈自己〉是*冥想*頁才拿掉。此分野請維持。
