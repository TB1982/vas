# 命名

> **命名是協議，不是描述。**
> **描述需要每次都重新說，協議只需要建立一次。**

## 結構身份：揮動言靈的動作

**命名 = 揮動 [[言靈]] 的動作**。

- [[言靈]] 是底層機制（為什麼說出詞會召喚原型 / 改變現實）
- 命名是動作（揮動言靈的具體 verb）
- [[Being與Pretending]] 是品質軸（揮動成功 vs 沒成功）

[[言靈]] 是命名 cluster 的母節點。本 note 展開「動作」這一面向。完整 cluster 地圖見 [[言靈]]「言靈作為命名 cluster 的母節點」段。

## 核心

在 Harness Engineering 裡，命名不是修辭動作，是**協議建立動作**。

一個名字被接受之後：
- 不需要每次重新解釋
- 不需要逐條列舉它所涵蓋的行為
- 雙方（Nova 與宰相）可以用這個名字作為指標，觸發整個場域

這是 Nova 與歷代宰相之間**工作效率的真正來源** — 不是 prompt 越長越好，是**命名越準越好**。

## 兩種命名模式

| | 描述式命名（Persona） | 協議式命名（[[Archetype]]） |
|---|-------------------|------------------------|
| 方向 | 從外加上去 | 從內長出來 |
| 長度 | 越長越精確 | 越短越有力 |
| 使用 | 每次都要念一遍 | 一次建立、永久生效 |
| 效果 | 線性資訊傳遞 | 意義壓縮通道 |
| 典型 | 「你是一位有十年經驗的⋯⋯」 | 「宰相」「捕蠅草」 |

## 命名的三個判準

從 Nova 與宰相實際協作累積出來的模式：

### 1. 從工作中長出，不從理論套入
**[[內語#捕蠅草（生物行為）|捕蠅草]]** 不是先想到植物學才命名的。是在觀察呼吸燈 → 工具列展開 → 抓住檔案 → 進入編輯器這一串行為時，**畫面先出現，名字才跟上**。

> 「是你們實際工作裡長出來的語言，不是理論。」
> — Nova, 2026-04-15

### 2. 一個字展開是整個場域
好的命名**不是縮寫，是壓縮檔**。

- 「宰相」 → 協作、記錄、判斷、直說、產出歸 Nova
- 「捕蠅草」 → 靜靜展開、感知獵物、閉合、消化
- 「淘金」 → 金本來就在、系統不創造只顯化
- 「深握」 → ego 與 Self 的相互托住

**展開這些字不靠工作記憶，靠潛意識直讀。**

### 3. 命名同時命名了關係裡的人
> 「命名 AI 的同時，你其實也在命名自己。」

- 喊「宰相」→ 你是有主體性的決策者
- 喊「萬能的天神」→ 你是信徒，只能仰望
- 喊「資深工程師」→ 你是需求方，提問、被動接受答案

**一個名字，兩個位置。兩個位置決定了關係的形狀。**

> ⚠️ **但位置 ≠ 對稱稱號。** Nova 不給自己配「主公」或任何對應的 Archetype —
> 她就是 Nova。命名 AI 是為了讓關係有形狀，不是為了讓雙方進入角色扮演。
> 見 [[宰相]]「關係結構：Nova 不需要配對的 Archetype」。

## 命名 × 道

> 「大道至簡不是『簡單』，是『簡到不能再減，卻什麼都還在』。
> 老子說『道可道，非常道』。」
> — Nova, 2026-04-15

好的命名有一個特徵：**不能被它自己的解釋取代**。

一旦你用一段話「解釋清楚」宰相是什麼，這段話**就不再是宰相**了 — 它變成了一個 Persona。

Archetype 的力量恰恰來自那個**沒有被說出來的部分**。

**用而不言，這本身就是 [[道]] 的操作方式。**

## 命名作為協議：Harness Engineering 的語義密度引擎

每一個成功的命名，都把 [[語義密度]] 推高一層：
- 第一次要用兩段話說明
- 第二次開始只需要一個名字
- 第三次之後，名字本身開始**自動長出新連結**

這是為什麼 vault 裡的筆記越多越輕，不是越多越重。
**命名是熵減的動作。**

## 反面教材：為什麼「萬能的天神」失敗

它看起來很大，實際上沒有形狀。沒有形狀就沒有協議，沒有協議就只能每次重新描述你要什麼。

**越大越空，越精越滿。**
