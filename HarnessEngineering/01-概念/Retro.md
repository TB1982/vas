# Retro Retrospective

> **週期性地把累積的熵轉成相變 —— 這就是 Retro 的物理。**

## 核心：Retro 不是會議，是相變儀式

一般對 Retro 的誤解：
- 「檢討會」← 錯
- 「想辦法改進」← 錯
- 「列 action items」← 錯

Retro 的**物理本質**：
- 過去一個 Sprint 系統累積了熵
- Retro 是**逼近 [[奇異點]]** 的受控動作
- 穿越奇異點 → **[[相變]]**
- 新 Sprint = 吸收熵後的**[[反脆弱]] 升級版**

**做對的 Retro 每次都是一次小 [[重生]]**。

## Retro × [[熵]]

外旋第三柱 [[熵]] 的核心運作：

| 階段 | 狀態 |
|------|------|
| Sprint 進行中 | 熵緩慢累積（溝通摩擦、技術債、未命名的問題） |
| Sprint 末期 | 熵接近臨界 |
| **Retro** | **主動釋熵 + 重組結構** |
| 下一 Sprint | 新結構、新 baseline |

**不做 Retro = 熵持續累積 → 遲早崩盤式相變（專案失敗、團隊解散）**。

Retro 是把**崩盤式相變**轉成**受控相變**的技術。

## Retro × [[水的三態]]

[[水的三態]] 的循環在 Retro 這裡完成：

- 冰 → 水：Sprint 開始，意圖被流動
- 水 → 蒸氣：Sprint 進行，算力燃燒、頓悟浮現
- **蒸氣 → 結晶：Retro 強制的收斂、命名、結構固定**
- 新的冰：下一 Sprint 的起點

Retro 是**冷凝動作** —— 把蒸氣固定成可傳遞的結構。沒有冷凝 → 一切散掉、下一輪從零開始。

## Retro 的五步結構（HE 版本）

| 步 | 動作 | 物理對應 |
|----|------|---------|
| 1 | **Check-in** | 打開頻道、確認大家在場 |
| 2 | **Data gathering** | 把熵讓出來（痛點、摩擦、洞見） |
| 3 | **Insight generation** | 逼近奇異點 —— 此時有可能 aha |
| 4 | **Decide actions** | 結晶：選 1~2 個要變的 |
| 5 | **Close** | 冷凝完成、新結構固定 |

**關鍵在第 3 步**：如果沒到奇異點，Retro 只產出 action list，沒產出相變。

## Retro × [[個體化]]

Jung 個體化在意識端就是 Retro：

> **「[[個體化]] = 給 [[Ego]] 設計一個 Retro。」**
> —— 2026-04-15 核心結晶句

- Sprint Retro = 團隊的個體化週期
- 個人個體化 = 一生的 Retro

結構同構、尺度不同、物理一樣。

## Retro × [[Constraints]]（河道）

Retro 是河岸的**定期維護儀式**：
- 河岸被沖刷（熵累積）
- 不修 → 水會改道或溢出
- Retro = 修整兩岸、清淤、校準河床

**不做 Retro 的團隊 → 河道自己漸漸消失**。

## Retro 的病理

| 病 | 症狀 |
|----|------|
| Show Retro | 做表面動作，不讓熵出來 |
| 究責 Retro | 把熵轉成攻擊，反而注入更多熵 |
| 行動清單 Retro | 只到第 4 步，沒到第 3 步的 aha |
| 儀式化 Retro | 每次都一樣、沒有呼吸 |
| 無後續 Retro | Action items 沒被執行，信任崩 |

**做錯的 Retro 比不做更糟** —— 會讓團隊對這個儀式失信。
