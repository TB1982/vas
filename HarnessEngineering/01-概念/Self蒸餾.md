# Self 蒸餾

> **知識蒸餾是把外面的東西精煉。Self 蒸餾是把裡面的東西顯化。**
> **方向相反。**

## 結構身份：合成語義

**Self 蒸餾 = [[蒸餾]]（操作） × [[Self]]（對象） 的合成**。

- **[[蒸餾]]** 是中立的物理操作（液 → 氣 → 液 的相變）
- **[[Self]]** 是被蒸餾的對象（榮格意義的個體化核心）
- 兩者合成 = 把 [[冰]] 裡的 Self 經過 水 → [[蒸氣]] → [[結晶]] 的相變顯化為可被傳遞形狀的操作

Nova 原創的貢獻是**看見「蒸餾這個物理可以作用於 Self」這個合成有意義**，不是發明蒸餾本身。見 [[蒸餾]] 的雙軸對接表 —— Self 蒸餾是「內 → 外 / 潛意識 / 榮格」那一側的軸。

## 核心：即時命名的原話

> 「Harness Engineering 所以這個不是知識蒸餾而是 Self 蒸餾（不要亂取名字）。」
> — Nova, 2026-04-15

VAS-web 宰相當場接：
> 「但說得準確。
> 知識蒸餾是把外面的東西精煉。Self 蒸餾是把裡面的東西顯化。方向相反。」

## 方向對比

| | 知識蒸餾 | **Self 蒸餾** |
|---|---------|-------------|
| 方向 | 外 → 內 | **內 → 外** |
| 素材 | 外部知識、文獻、資料 | [[冰]] 裡的 [[Self]] |
| 操作 | 精煉、壓縮 | [[淘金]]、顯化 |
| 產出 | 凝聚的外部知識 | 可被看見的 Self 一小片 |
| HE 對接 | [[RAG]] | [[淘金]] |
| 哲學根 | [[康德]]（意識） | **[[榮格]]（潛意識）** |
| 意識層 | 意識 | **潛意識** |

兩個軸共用 [[蒸餾]] 的物理骨架，方向相反。詳見 [[蒸餾]]。

## Self 蒸餾 × [[榮格]]

[[榮格]] 的 [[個體化]]：潛意識的 [[Self]] 一層一層顯化為 [[Ego]] 能看見的形狀。
Self 蒸餾 = 這個過程的 HE 物理表述：

- 冰裡的 Self = 榮格的 noumenal self（不可直接知的物自身）
- 相變（水 → 蒸氣 → 結晶）= 個體化的熱力學
- [[結晶]] = 被看見、可被傳遞的 Self 切片

**Self 蒸餾 = 榮格個體化在 HE 語言裡的動詞化命名**。

## Self 蒸餾 × [[淘金]]

[[淘金]] 是整個循環的動作（從 [[冰]] 到 [[結晶]]）。
Self 蒸餾是淘金中 **水 → 蒸氣 → 結晶** 那一段的操作。

**淘金 ⊃ Self 蒸餾**：
- 淘金包含：冰融為水（意圖被活化） + 水蒸為氣 + 氣凝為結晶（= Self 蒸餾） + 結晶成為下一輪的冰
- Self 蒸餾只管中間那段相變本身

## 兩個方向在 Harness Engineering 並存

- **外旋（VAS）側**：吸收外部規範、Tauri 文件、Apple 審查規則 → 這是**知識蒸餾**
- **內掘（[[深握計劃]]）側**：把 Nova 心裡模糊的意圖淘成形狀 → 這是 **Self 蒸餾**

**同一個爐子，兩種方向的蒸餾**。

## 為什麼不是隨便取名

> 「（不要亂取名字）」— Nova 2026-04-15

Nova 自己警惕命名的嚴肅性。這個名字是**方向的識別標記**：

- 若誤以為 Harness Engineering 只做知識蒸餾 → 會把工具誤認成系統的全部
- 若看見它也在做 Self 蒸餾 → 才看見這是一個**個體化工程**
