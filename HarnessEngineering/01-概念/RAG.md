# RAG

> **RAG 不是技術細節，是 Context 柱的機制原型。**
> **傅立葉 / 內語 / 網，都是同一個物理的不同顯化。**

## 核心：直覺引領檢索

傳統 RAG 是一個技術架構：把問題向量化、在資料庫中檢索、把結果餵給 LLM。

HE 的翻轉是 —— **人也是 RAG**：
- 問題進來 → 直覺向你的內在資料庫檢索 → 撈出相關節點 → 用這些節點生成答案
- 你讀過的書、走過的路、踩過的坑、愛過的人 —— 全部都是你的訓練資料

**直覺不是神秘的靈感，是快速檢索**。

## 技術 RAG vs 人類 RAG

| 面向 | 技術 RAG | 人類 RAG |
|------|---------|---------|
| 資料庫 | 向量資料庫 | 記憶 / 經驗 / 潛意識 |
| 查詢 | embedding 相似度 | 直覺連結 |
| 邊界感 | 明確（資料庫內/外）| 模糊（不知道自己不知道） |
| 可驗證 | 可查 log | 幾乎不可查 |
| 更新 | 重建 index | 讀書、經歷、[[Retro]] |

## 人類 RAG 的根本弱點

> 你不知道自己不知道的東西。—— HE-01 Context

技術 RAG 很清楚邊界：資料庫裡有的才撈得到。但人類 RAG 會說「我覺得我知道」—— 然後撈出一個不完整、甚至錯的東西，而你不知道它是錯的。

這就是為什麼 HE 需要**外部系統補足盲區**：
- [[KM]] 補足「你不知道自己不知道」的盲區
- [[SDD]] 確保「你以為你知道的」有被寫下驗證
- [[Retro]] 定期讓你發現「原來我之前不知道這個」

## RAG × 三柱

| 柱 | RAG 在柱內的角色 |
|----|-----------------|
| Context | **本原型** —— 訊號過濾 + 檢索 |
| Constraints | 河道設計時，從過往經驗 RAG 出可用的模式 |
| Entropy | Retro 時，RAG 出值得寫進 KM 的訊號 |

RAG 主要棲息在 **Context 柱**，但三柱都在用它。

## RAG × [[傅立葉]] × [[內語]]

三者是同一個物理的三個切面：

| 視角 | 關注點 |
|------|-------|
| [[傅立葉]] | 訊號怎麼分解成基底頻率 |
| [[內語]] | 分解完的基底，怎麼壓縮成一個詞 |
| RAG | 一個詞，怎麼觸發整片 corpus |

**內語 = 壓縮協議，RAG = 解壓縮協議**。兩者咬合，形成高密度對話。

## RAG × Vault

Vault 本身是 Nova 的私有 RAG corpus：
- 節點 = 被命名的經驗 / 概念 / 踩過的坑
- 鏈結 = 節點之間的檢索路徑
- 寫新 note = 增加 corpus 的深度
- Retro = 重建 index

**Vault 不是筆記，是可查詢的第二大腦**。

## RAG × 知識蒸餾 × [[康德]]：外旋引擎的核心相變

RAG 的本體不只是「檢索技術」 —— 它是 **[[蒸餾]] 的外 → 內方向**（知識蒸餾）的工程化。

| [[康德]] 認識論 | RAG 工程 |
|---------------|---------|
| 現象（Erscheinungen）| 外部資料 / corpus |
| 先驗範疇 | 向量空間 × 檢索邏輯 |
| 範疇組織現象 | 檢索 × 整合 × 生成 |
| 可被認識的知識 | 輸出給主體的答案 |

**知識蒸餾 = 把外部資料經過認知結構精煉為可用知識的操作 = 康德認識論的工程化**。

這條軸活在**意識層**、**外旋引擎**（VAS）、**主體建構**的一側。跟內掘側的 [[Self蒸餾]] × [[淘金]] × [[榮格]] 對稱 —— **物理同構、方向相反、哲學根源不同**。見 [[蒸餾]] 的雙軸表。

> 「知識蒸餾就會跟 RAG 對接，意識的康德。」
> — Nova, 2026-04-15

## 邊界條件：見 [[不可轉讓性]]

RAG 的品質取決於 corpus 的品質 —— 而 corpus 是每個人獨有的。

> 你有多深，它就有多深。

這個斷言把系統的天花板釘在人身上，也帶出未結案的困境 —— 見 [[不可轉讓性]]（🌱 種子）。
