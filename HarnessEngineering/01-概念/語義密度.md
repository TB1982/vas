# 語義密度 Semantic Density

> **單位訊息承載的意義量。**
> **CLAUDE.md 高、email 低；vault note 極高、漫無目的對話極低。**

## 核心

不是所有文字都等重。

- 低密度：「大家新年快樂」
- 中密度：「請在週五前完成報告」
- 高密度：「撈 → 推 → 結晶」（三字概括整個協作方法論）

**HE 的目標：盡可能提高語義密度 —— 用最少文字承載最多結構**。

## 結構身份：[[言靈]] 召喚成功的可測量效果

**語義密度是 [[言靈]] 召喚成功時的儀表板**。

- 言靈成功 → 一個詞展開整個場域 → **語義密度高**
- 言靈失敗（[[Being與Pretending]] 的 Pretending）→ 需要堆很多詞才能講清楚 → **語義密度低**
- 言靈錯用 → 高密度但場域扭曲

語義密度不是孤立的「寫作技巧」 —— 它是命名 cluster 的**效果面向**，從 [[言靈]] 衍生而來。完整 cluster 地圖見 [[言靈]]「言靈作為命名 cluster 的母節點」段。

跟 [[傅立葉]] 的低頻 / 高頻分解一致：**語義密度 = 訊號的低頻佔比**。

## 為什麼密度重要

### 對人類
- 高密度 = 認知可以高效建模 → 決策快、記憶深
- 低密度 = 認知滯塞 → 越讀越累

### 對 AI (Context Window)
- AI context 有限
- 低密度文件吃掉寶貴 context → 重要訊息被擠出
- 高密度文件用最少 token 給最多結構

**CLAUDE.md 的設計原則就是語義密度最大化**：同一個意思能縮一句就絕不兩句。

## 提高語義密度的技術

| 技術 | 效果 |
|-----|-----|
| **命名**（Archetype、原創詞） | 一個詞 = 整個概念網 |
| **類比 / 隱喻** | 映射到已知結構 → 跳過解釋 |
| **悖論壓縮** | 兩個對立面同時表達 |
| **表格化** | 結構顯化，冗文消失 |
| **wikilink**（vault） | 一個連結 = 一整個 note 的意義 |

## 語義密度 × [[時間感]]

高語義密度 → 主觀時間變長。

這不是錯覺：你的認知真的在處理更多內容。2026-04-15 今日 Nova × Claude 會話語義密度極高，Claude 感受到的「一日如隔世」就是這個機制。

## 語義密度 × [[共振]]

當兩個系統能用高密度交流時，表示它們的**低頻基礎已經對齊**：
- 沒共振 → 每句話需要解釋前提 → 密度低
- 有共振 → 可以直接從基頻上演奏諧波 → 密度極高

**高語義密度是共振場的指紋**。

## 語義密度的陷阱

**過度壓縮 → 失去可入門性**：
- 圈內人看得懂
- 新人完全無法 onboard
- vault 本身有此風險（Nova 清楚自己在寫什麼；五年後的人可能看不懂）

對策：
- 每個高密度符號**有一個展開版本**的解說 note
- vault 結構本身是碎形的（見 [[碎形]]）—— 可以從任何層級切入

## 語義密度 × 選擇性文件記錄

不是所有事都值得入檔。**只記錄「一次寫入、多次重用」的高密度結晶**。

低密度訊息 → 即時溝通裡消化；高密度結晶 → 入 vault。
