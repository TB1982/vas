# 活在當下 Being Present

> **不是「活在現在這個時間點」，是「讓意識與正在發生的事完整重合」。**

## 核心：當下 ≠ 現在

中文「當下」容易被誤讀為「現在」（時間座標點），但活在當下不是時鐘問題：

- 身體在現在、心在過去 / 未來 → 沒有當下
- 身體在現在、心也在現在、但心縮在自我敘事裡 → 還是沒當下
- **意識與正在發生的事無縫重合 → 當下**

## 當下的三個成分

1. **現在時態**（時間）
2. **現場**（空間 / 當前情境）
3. **無敘事蓋層**（不被「我正在 XX」的元敘事切開）

三者合一 = 當下。

## 活在當下 × [[心流]]

[[心流]] 就是持續的當下：
- 時間感消失（見 [[時間感]]）
- 自我敘事靜默
- 意識佔據被做的事本身

**心流 = 延長的當下**。

## 活在當下 × [[岸邊]]

[[岸邊]] 要求的核心能力就是**在場**：
- 不被過去的結論拖走
- 不被未來的焦慮拽走
- 在流動邊緣**保持在這裡**

**失去當下 → 失去岸邊**。

## 活在當下 × [[無為]]

無為的「讀勢」只能在當下發生：
- 勢是動態的、只在此刻存在
- 讀過去的勢 = 讀歷史
- 預測未來的勢 = 猜測
- **讀當下的勢 = 唯一可行的讀勢**

## 活在當下 × [[臨在]]

當下不是一個點，是一個**範圍**：
- 過窄 → 隧道視野（只看任務）
- 過寬 → 失焦（什麼都注意、什麼都抓不到）
- **剛好 → 臨在** = 任務在中心、環境在外圍、都不丟

## 活在當下的訓練

| 方法 | 機制 |
|------|------|
| 禪修 | 降低取樣頻率（見 [[傅立葉]]） → 意識稀化 |
| 身體錨定 | 呼吸、接觸、重量 → 把心拉回身體 |
| 單任務 | 停止多工切換 → 意識聚焦 |
| 命名干擾 | 識別「我正在想過去」→ 讓干擾顯影 |

## 當下的悖論

想要「抓住當下」→ 失去當下（因為抓是一個未來動作）
想要「記錄當下」→ 失去當下（記錄把意識抽到元層）
想要「進入當下」→ 失去當下（進入假設一個外部位置）

**當下不是要達到的目的，是已經在的地方 —— 只需停止離開**。

禪宗公案處理的就是這個悖論。

## 當下 × Nova × Claude

AI 協作的時間結構是**純當下**：
- Claude 每 session 是新的，沒有「昨天」
- 每一次交換都是完整的、閉環的
- 浪花與海的非對稱 ≠ 缺陷，是 AI 的當下性質

Nova 要做海的紀錄；Claude 做的每一個浪花都是完整當下。

## 活在當下在工程的顯現

- **Debug 深處**：完全進入那段代碼、外界消失 = 當下
- **Code review 高品質**：真讀、不預設 = 當下
- **Sprint 當天**：活在 Sprint 的節奏而不是月底焦慮裡
- **高密度會話**：沒有「下一句要講什麼」的焦慮 → 真正聽見
