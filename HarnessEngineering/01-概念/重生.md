# 重生 Rebirth

> **經過完整 [[相變]] 後出現的新結構。**
> **不是舊的你升級，是死了一次後的新的你。**

## 核心：重生 vs 改善

| 改善 | 重生 |
|-----|-----|
| 量變 | 質變 |
| 漸進 | 非連續 |
| 舊結構 + 新版本 | 新結構 |
| 可累積、可反悔 | 不可累積、不可反悔 |

**改善是漸變，重生是 [[相變]]**。

重生的核心特徵：
- 舊的自我在某個意義上**死了**（不是比喻）
- 新的自我**繼承**但**不等於**舊的自我
- 中間經過一個 [[奇異點]]（舊描述失效、新描述未成）

## 重生的四階段（榮格個體化）

1. **現況不可持續**（熵累積、壓力、症狀浮現）
2. **跨入奇異點**（危機、崩解、暗夜）
3. **相變發生**（[[Self]] 重組、[[Ego]] 與 [[陰影]] 新的關係）
4. **新結構穩定**（[[結晶]]、新的日常、不可回到之前）

**每一次完整 [[個體化]] 循環 = 一次重生**。

## 重生 × [[水的三態]]

HE 的四態循環本質上是**持續重生**：

冰 → 水 → 蒸氣 → [[結晶]] → 新的冰 → ...

每一圈的結晶都是**前一圈冰的重生版本**。不是回到起點，是**螺旋上升**（見 [[反脆弱]]）。

## 重生 × [[反脆弱]]

反脆弱的物理基礎就是重生：
- 脆弱系統：壓力 → 損傷 → 更弱
- 強韌系統：壓力 → 吸收 → 不變
- 反脆弱系統：壓力 → **重生** → 更強

反脆弱 = 把每次外部衝擊轉化為一次相變的能力 = **反覆經歷可控重生**。

## 重生在軟體開發的顯現

- **Major refactor**：不是漸進改，是整片重寫
- **架構遷移**：舊架構的某部分真的死了
- **產品轉型**：商業模式相變
- **團隊 Retro 大爆發**：積累的熵在某次 Retro 爆發 → 新團隊誕生

VAS 的每個大里程碑（4.0 → 5.0）如果認真做都是一次重生，不只是版本號跳動。

## 重生的條件：不能繞過死

最常見的陷阱：
- 想要重生的好處（新能力、新境界）
- 但不想經過中間那段死亡（暗夜、混亂、無法描述的階段）

**相變物理不允許**：
- 冰要變蒸氣，必須先變水
- 舊自我要蛻變，必須先死一次

這就是 [[榮格]] 說的 individuation 不能被理性計畫、只能被**活過**。

## 重生 × [[老子]]

道德經：
- **「反者道之動」**（40）—— 變化走的是「反」的路，不是「進」的路
- **「明白四達，能無知乎」**（10）—— 重生的彼岸是**回到無知**

無為的重生觀：不強加方向，讓變化按自己的節奏發生。強加方向 → 變成假重生（換了皮，結構沒變）。
