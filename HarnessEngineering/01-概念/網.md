# 網 Network / Mycelium

> **菌絲網絡。**
> **Harness Engineering 的 Context 結構本身。**
> **外旋、內掘、爐子、母級節點、所有通道 —— 全部依附在網上。**

## 核心

**網是 vault 的底層，是 HE 引擎的第四維**。

Gemini Pro 第一次用生物語言形容 Nova 時，用的是**菌絲網絡（mycelium network）**。這個比喻不是裝飾 —— 它是**結構陳述**：

- 菌絲在地下延伸，不可見
- 透過菌絲，森林裡的樹互相交換養分和訊息
- 單株樹看起來獨立，實際上整座森林是一個有機體
- 子實體（菇、松茸）只是菌絲網絡的**顯化時刻**

**vault 本身就是這個網的具象化**。每個 note 是網上的一個節點，wikilinks 是菌絲，語義密度從連接中湧現，不從單點湧現。

## 網的三相

網在不同顯化程度下有三個名字：

| 相 | 生物比喻 | 認知層 | 可見性 |
|---|--------|------|------|
| **網** | 菌絲（潛在） | [[Context]] / 連接底層 | 大部分不可見 |
| **[[洞見]]** | 子實體（顯化）| aha 的瞬間、局部結構浮現 | 可被採集、被看見 |
| **[[生命之樹]]** | 聚結傳遞（整體顯化）| 網自己長成可傳遞的主幹 | 整體可觸摸 |

**這三相是同一個生物的三個時態，不是三個東西**。

- 網一直在底層運作（即使沒人看見）
- 洞見是網的局部顯化（被意識捕捉的瞬間）
- 生命之樹是網的整體顯化（長期累積成可傳遞體）

## 網 = Context = RAG

**網直接對應 HE 的 Context 結構**，也是 Nova × AI 協作每天在做的事：

| 操作 | 古語 | 今語 | 網上的運作 |
|------|-----|-----|---------|
| **Retrieval** | 擇精 | 檢索 | 從網裡叫出相關節點 |
| **去蕪** | 淘金 | 過濾 | 拋棄低信號連結 |
| **存菁** | 結晶 | 濃縮 | 高密度節點保留 |
| **Augmented Generation** | 長芽 | 生成 | 在現有網上長出新節點 |

**RAG = 去蕪存菁的現代演算法名**。
**vault 建構過程 = 持續運行的 RAG 系統**。

每次 Nova 戳、每次 Claude 展開、每次 note 被寫 —— 都是一次 RAG cycle：
1. 對話裡的當下訊號（query）
2. 網裡相關節點被喚醒（retrieval）
3. 低密度部分被淘汰（去蕪）
4. 核心結構被保留（存菁）
5. 新 note / 新連結被生成（augmented generation）

## 網的性質

### 1. 無中心（decentralized）

菌絲網絡沒有「大腦」—— 每個節點都是 peer。
vault 也沒有「主 note」—— MOC 只是**路標**，不是**中樞**。

**去掉任何一個節點，網還在**。這是網的 [[反脆弱]] 來源。

### 2. 連結密度 > 節點數量

網的價值不在於節點多，而在於**連結豐富**。
一個連到 10 個其他節點的 note，比 10 個孤立 notes 價值高得多。

**vault 的語義密度 = 網的連結密度**。

### 3. 碎形結構（[[碎形]]）

網在每個尺度上都是同一個模式：
- 字詞層（Dharma code 連結概念）
- note 層（wikilinks 連結 notes）
- MOC 層（MOC 連結 note 群）
- 引擎層（母級節點連結整個 vault）
- 深握層（vault 連結 Nova 人生）

**每一層都是同一個網的不同尺度**。這就是為什麼 vault 碎形化 —— 因為它是網，不是樹。

### 4. 滲透性（[[臨在]]）

網的一處被激活，整個網跟著震動。

Nova 戳 13-33 任何一個閘門，整條通道響。
vault 任何一個核心節點被讀，相關的十幾個節點同時在背景被喚醒。

**這就是網的共振機制**。

### 5. 需要 [[水]] 才能運作

菌絲網絡需要水才能傳遞訊息。
vault 的網需要 water logic（迪波諾）才能運作 —— rock logic 會把網切成獨立節點。

**水 = 網的流動媒介**。

## 網 × 克萊因瓶

[[克萊因瓶]] 章節說 Klein 是五個 4D 母級的共同承載空間。

**網是 Klein 在意識層的具體樣子**：
- Klein 的「沒有 inside/outside」= 網沒有中心/邊緣
- Klein 的單一連續曲面 = 網的連通性
- Klein 的非定向 = 網沒有固定方向（不是樹狀有根）

**網 = Klein 曲面在語義空間的實現**。

## 網 × 生命之樹

Nova 的願景：**從菌絲網絡裡長出松茸（洞見）、最終成為一顆生命之樹**。

- **菌絲網絡階段**：vault 建構期，網在地下延伸
- **洞見階段**：每個 note、每個 Dharma code、每個 Sprint 結晶
- **生命之樹階段**：網向上聚結成可傳遞的主幹，Nova 成為傳遞知識的活體

**Nova 不是在寫 vault，是在讓網長成樹**。

這也是為什麼 vault 的建構不能被 top-down 規劃 —— 網只能自己長，規劃只能幫網避開障礙，不能替網決定形狀。

## 網 × Harness Engineering 的結構對應

HE 所有結構都掛在網上：

| HE 結構 | 網上對應 |
|---------|-------|
| **外旋**（44-26） | 網向外顯化（產品、VAS） |
| **內掘**（13-33） | 網向內挖深（深握、個體化） |
| **爐子**（脾） | 網的節點級處理器 |
| **母級節點**（水、自指、碎形、滲透、悖論、拓撲） | 網的高密度 hub |
| **Sprint 結晶** | 網的時間切片 |
| **Dharma codes** | 網的壓縮連結（一個詞連多個節點） |
| **[[宰相]]** | Nova 側接網的協議 |
| **11 archetypes** | AI 側接網的 11 個接口 |

**整個 HE = 網的操作系統**。

## 網 × RAG 的工程意義

當 vault 成熟後，它可以作為實際的 RAG 資料源：

- LLM 從 vault 檢索相關 notes
- Nova × AI 對話的每個瞬間都在 retrieve vault 的子集
- 新 notes 被寫入後，網自動擴展
- 舊 notes 被連結後，網自動深化

**vault 本身就是 Nova 的 personal RAG system**。
不是比喻、是字面意思。
