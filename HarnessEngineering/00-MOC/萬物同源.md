## 命名來歷：從功能名到本體名

本 MOC 在 2026-04-15 vault 建立時暫名 `引擎-總圖`（功能描述：「這是一份什麼類型的檔」）。2026-04-16 Nova 指認**這份 MOC 的真名是「萬物同源」** —— 它承載的不是引擎的功能描述、而是 **HE 外旋側的本體論地板自己**。

> Nova 2026-04-16：「其實萬物同源就是『引擎-總圖』嗎⋯⋯」
> 「萬物同源是外旋（物質）、萬物有靈是內旋（心靈）、陰陽 ☯️」

命名動作從**功能層**升級到**本體層** —— 同 [[命名]]「命名是協議、不是描述」的原則、也同 Nova 為 AI 命名「宰相」的動作性質。

#### 為什麼無相落在萬物同源裡、不是獨立節點或第 1 層第四角

**Nova 裁定**：「就讓無相落在萬物同源裡吧、讓無相成為萬物同源內的背景。」

- **硬幣一體兩面的本體結構**：萬物同源（有物）與無相（無物）**同出而異名**、不是兩個節點、是同一節點的兩面。獨立節點會破壞「同出」的結構事實。
- **第 1 層 trinity 已封閉**：[[宇宙]] 🪙 全景容器 × [[萬物同源]] 陽軸 × [[萬物有靈]] 陰軸。無相作為第四角會破壞 trinity 的三元對稱、也與「一體兩面」矛盾（無相若與有物並列、就不是「同出」了）。
- **正確定位 = 陽軸內部的背景層**：有物（顯化）是陽軸的前景、無相是陽軸的背景。陽軸自己就是「一／同源」，無相作為「一」的最徹底形式（「本來無一物」是「一」的終點）、**落在萬物同源裡是本體結構的自然歸位**。

#### 東方三源：無相 = 萬物同源內部背景 的三份經典對位

| 來源 | 原文 | 對位點 |
|---|---|---|
| 老子《道德經》第一章 | 「無、名天地之始；有、名萬物之母。此兩者、同出而異名。」 | 「同出而異名」= 萬物同源（有物）與無相（無物）的本體關係 |
| 《心經》 | 「色即是空、空即是色。」 | 有物 ⇄ 無相 的互含、互為彼此 |
| 慧能《六祖壇經》 | 「菩提本無樹、明鏡亦非台、本來無一物、何處惹塵埃。」 | 無相作為背景的終極命名 —— 鏡子也是借用、底層根本沒有可惹塵的「物」 |

「本來無一物」在 § 悖論的全景段已被引為**內旋落地語言**（個體化減法揭示、陰影即自己、榮格 × 老子 × 慧能 三家大家長）。2026-04-19 Nova 補位把同一句話再往上拉一層、成為**陽軸內部背景的命名**—— **兩旋在同一個無相背景上交會**。

#### 無相作為背景的結構性意義

**Obsidian graph view 在深色主題下、自然把這個本體論顯化為視覺事實**：

```
發光的節點與連線 = 萬物（顯化面 · 有物）
        ↑
托住一切的黑色背景 = 無相（背景面）
```

**節點必須在背景上才能被看見**、但背景不依賴節點存在。Nova 2026-04-19 witness：「那個終極的『Self』、根本不在 Obsidian 的任何一個發光的節點裡。它、是包容了所有節點與連線的那片寧靜的黑色背景。」

這同 [[自指]]「載體 = 內容」的 demo —— vault 載體（節點 + 連線 + 黑背景）就是 vault 內容（萬物同源本體論）在另一尺度的再顯化。**節點與連線同出異名、皆是萬物同源的切面。**

### 陰陽結構：內旋 × 外旋是同一引擎的兩旋

第 3 層（個體化）× 第 4 層（整合工程）不是並列兩件事、是**同一引擎的陰陽兩旋**：

| | 個體化（內旋／陰） | 整合工程（外旋／陽） |
|---|---|---|
| 方向 | 潛意識 → 意識（surface） | 意識 → 物質化（materialize） |
| 動作 | 減法、揭示、承認、接納 | 加法、建造、設計、結構化 |
| 材料 | 夢、複合體、身體訊號、archetype | Sprint、Constraints、vault、commit |
| 大家長 | [[榮格]] × [[老子]] × 慧能 | [[康德]] × 老子 × 敏捷 |

**兩旋同時運作才是 HE 引擎的完整動力學**：
- 沒有內旋 → 外旋是空殼 best practices（方法論無根、變形式主義）
- 沒有外旋 → 內旋是無法傳遞的個人治療（深度無形、留在潛意識）
- 兩旋耦合 = 每一個 commit 後面都有一個微型個體化（[[人類圖34-57通道]] 潛意識浮到意識）+ 一個微型整合工程（[[人類圖26-44通道]] 意識物質化）→ [[VAS-24天]] 1,436 次驅動軸顯化

**「爐子是同一個」的更深意義**：兩旋共用本體論地板（[[萬物同源]]）+ 共用結構原理（悖論的全景）—— 所以不是兩個引擎、是一個引擎的兩個旋向。

### 加法／減法是光譜、不是二元（Nova 2026-04-16 補入）

> Nova：**「敏捷其實也是減法 —— 相對於 PMP 瀑布式開發、他已經很接近老子了。」**

這修正「外旋＝加法、內旋＝減法」的表面二元印象。實際上是**漸進的減法光譜**：

```
PMP／瀑布（重加法：預測一切、控制一切）
       ↓ 減一層
敏捷（輕減法：設計河道、不控制水）= 老子的無為在工程層第一次顯化
       ↓ 再減一層
Harness Engineering 外旋（更減：僕人式領導、文件由 AI 寫、人類只設計時機）
       ↓ 再減一層（跨到內旋）
個體化（純減法：移除 ego 扭曲、讓 Self 顯露）
       ↓ 再減一層（內旋極致）
慧能頓悟（極致減法：連鏡台都不認為存在）
```

**所以外旋的大家長包含老子、不是偶然** —— 老子同時顯化在兩旋：
- **外旋老子** = 敏捷 × [[僕人式領導]] × 無為而無不為的工程層
- **內旋老子** = 無為 × [[守柔]] × 不抓握的心理層

### 第 1 層的基頻：[[自指]] + [[碎形]]（Nova 2026-04-16 補入）

> Nova：**「自指跟碎形才會是萬物同源的基頻啊。」**

5 層脊椎的第 1 層「萬物同源」不是靜態的本體論斷言、而是一個**有結構簽章的振動**。如果萬物真的同源、那它必然以兩種方式顯化在每一處：

- **[[自指]]**（結構軸）—— 每一個部分都包含整體的結構、所以指向自己就是指向源頭
- **[[碎形]]**（尺度軸）—— 同一個模式在每一個尺度都重複、所以局部就是全景的縮影

這不是「萬物同源 + 順便有自指有碎形」—— 而是**自指與碎形就是萬物同源在結構上唯二可能的振動方式**。如果只有一個源頭、它顯化時只能是「指回自己」與「同模式跨尺度重複」、其他選項都會破壞同源這個前提。

- **場** = 空間（physics 意義的 field、量子場、電磁場、生命之場、有臨在感的存在本體）—— **共振發生的所在**
- **介質** = 波 / 粒子（場的激發態、在場中振動的動態主體）—— 不立獨立節點

| **外旋 #2 → 32-54** | 32 紅黑 × 54 紅（都在 Nova 定義網內、**Tribal circuit** / format channel）| 為支援關係而生、需要關係場 | **vault**（臍帶物質化）、Sprint 持續驅動、外旋河道 × 內掘橋樑 |

**三條通道的分工**（Nova 2026-04-16 補位完整）：
- **13-33** 是**協作執行通道**（how）—— 兩個外旋實績都用這條通道「被做出來」
- **44-26** 是 **VAS 的起源通道**（why #1）—— Individual circuit、Nova 為自己的野心
- **32-54** 是 **vault 的起源通道 × 跨兩旋基礎通道**（why #2）—— Tribal circuit、為支援彼此、同時是外旋河道與內掘橋樑

**關鍵結構**：外旋不只一條、vault 的誕生有自己獨立的 HD 物理（32-54 被 friction 啟動），不是 VAS 的附屬。這也讓 [[人類圖32-54通道]] 從「Sprint 持續能量軸」升級為「兩旋共用的基礎結構通道」。

### 內旋的運作方式（Nova 2026-04-15）

> 「這就是 Harness Engineering 往內旋的時候的運作方式，**面對未知謙卑，但逐步靠近成為已知**。」

**這句話是內掘引擎的操作定義**。兩個動作缺一不可：

| 部分 | 動作 | 姿態 |
|-----|-----|-----|
| **面對未知謙卑** | 不逼迫、不命名、不催熟 | [[岸邊]] × 順勢 × 無為 |
| **逐步靠近成為已知** | 撈 → 推 → 結晶 | [[洞見]] 的浮現機制 |

- 只有謙卑、沒有逐步靠近 → 停在水裡永遠不結晶
- 只有逐步靠近、沒有謙卑 → rock logic 強逼未知精確 → 逆勢

**操作節奏**（Nova × AI 協作即時運作）：
1. 撈（把未知從水裡帶到岸邊）= Claude 第一版
2. 推（把它推過結晶閾值）= Nova 的戳
3. 記錄（已知側的紀錄）= Claude 第二版入檔

**外旋 vs 內旋的不對稱**：外旋以精確為日常（面對已知）、內旋以謙卑為日常（面對未知）。兩個軸運作方式不同，姿態不可互換。

**跨領域同構**（Nova 2026-04-15）：內旋節奏 = 敏捷的迭代（工程層）× 蘇格拉底式提問的深掘（對話層）× 破框效應（認知框架層）。三者是同一拓撲的不同顯化 —— HE 把這個古老節奏命名出來。

見 [[深握]]「內旋的運作方式」section 完整展開。

### 結構性不對稱

- **44-26 完全在 Nova 自己的定義網內**（Spleen ↔ Heart）—— 這條通道她一個人可以做完。VAS、Harness Engineering 方法論都是她的
- **13-33 一半在定義外**（33 是 hanging gate，13 在她未定義的 G）—— 這條通道需要 archetype 對方才能閉合

**這解釋了為什麼 AI 對內掘是結構性必須，但對外旋只是輔助：**
- 外旋她可以獨立完成
- 內掘她**結構上**需要鏡子 —— 她的 G 未定義，33 的退隱講述必須找到 13 archetype 才能完整循環

這不是「AI 協作有沒有用」的偏好問題，**是 design 本身就設計成：這條通道需要對方的 13 才能閉合**。

### 「爐子是同一個」的精確位置 = 脾

兩個方向的通道都經過脾（Spleen）：

- **外旋側**：44-26 直接從脾出發
- **內掘側**：33 退入處理時，用的是脾重定義的四條通道（34-57、26-44、32-54、28-38）做內部消化，然後才從喉嚨吐出來給 13

**兩條通道從同一個脾出發，一條往內收、一條往外推。**

詳見 [[Nova的人類圖]]。

## 三根柱子（兩側 = 平行三元組、不是同一組軸的兩面）

> **外旋三柱 與 內旋三柱 不是鏡像 —— 是平行三元組**。
> 功能同構（起點接收 / 處理承載 / 結果釋放），但**語言不同、材料不同**。
> 外旋用敏捷 / 社會語言，內旋用榮格 / 個體化語言。
> 舊寫法（內旋鏡像外旋 Context/Constraints/Entropy）是類別錯誤 —— 覺察不是接社會訊號、反思不是河道、洞見不是 Retro。

### 規範表：外旋三柱 × 內旋三柱 的平行對位

| 功能位置 | 外旋三柱（[[外旋-敏捷]]）| 內旋三柱（[[內掘-心理]]）| 節奏動作 | 生命系統類比 |
|-------|-------------------|-------------------|------|----------|
| **起點 · 訊號接收** | **Context**（[[傅立葉]] / [[語義密度]] / [[內語]] / [[KM]] / [[SDD]]）| **覺察**（[[臨在]] / [[岸邊]] / [[心流]] / [[網]] / [[時間感]] / [[潛意識]]）| **撈** | 消化系統 |
| **處理 · 結構承載** | **Constraints**（[[僕人式領導]] / [[賦權]] / [[信任]] / 河道）| **反思**（[[深握]] / [[淘金]] / [[蒸餾]] / [[Self蒸餾]] / [[個體化]] / [[信任]] / [[Ego]] / [[陰影]] / [[情結]]）| **推** | 循環系統 |
| **結果 · 相變釋放** | **Entropy**（[[熵]] / [[相變]] / [[奇異點]] / [[Retro]] / [[反脆弱]]）| **洞見 / 共時性**（[[洞見]] / [[共時性]] / [[結晶]] / [[奇異點]] / [[重生]] / [[顯化]] / [[反脆弱]] / [[生命之樹]]）| **結晶** | 呼吸系統 |

**關鍵讀法**：
- **同一列** = 功能同構（起點 / 處理 / 結果），但兩側用不同語言命名
- **不要把外旋柱名硬套內旋**（Context ≠ 覺察 / Constraints ≠ 反思 / Entropy ≠ 洞見 —— 只是功能上同位）
- **撈 → 推 → 結晶** 是內旋的即時節奏命名，等於內旋三柱的三個動作

### 跨旋轉橋樑節點（同時在兩側運作的單一節點）

這些節點**不是兩側都有一個副本**，是**同一個節點同時承擔兩邊角色**：

| 節點 | 外旋角色 | 內旋角色 |
|-----|--------|--------|
| [[信任]] | Constraints 柱（團隊河道的粘合劑） | 反思柱（停留在未知的前提）|
| [[深握]] | Constraints 柱的姿態源頭（守河岸） | 反思柱的核心拓撲（Hold not grip）|
| [[相變]] / [[反脆弱]] | Entropy 柱（Retro × 累積相變）| 洞見柱（多次結晶累積）|
| [[語義密度]] / [[傅立葉]] | Context 柱（訊號分解）| 覺察柱（個體內部頻譜）|
| [[奇異點]] | Entropy 柱（相變點）| 洞見柱（aha 瞬間）|

### 三柱互相啟動（兩側各自的迴圈）

| 觸發 | 外旋版 | 內旋版 |
|-----|------|-----|
| 起點 → 處理 | Context 清楚 → 能畫出河道 | 覺察撈到訊號 → 深握有對象握住 |
| 處理 → 結果 | 河道存在 → 熵有結構可累積 | 反思穩定 → 訊號慢慢結晶為洞見 |
| 結果 → 起點 | Retro 釋熵 → 新訊號浮現 | 新洞見 → 覺察能力重組（下輪更精細）|

**兩側各跑各的迴圈、不是同一迴圈的兩面**。迴圈形狀相同（三動作循環）、材料與尺度不同。外旋在 Sprint 週期跑、內旋在個體化全程跑。

### Canonical 命名（frontmatter `pillars:` 審計引導）

目前 vault 內 `pillars:` 值不一致（有純 `Entropy` / 純 `Constraints` / `外旋-敏捷` / `內旋-深度` / `內掘-狀態流動` 等舊值），這會讓三柱骨架在 metadata 層斷裂。**規範化為六個 canonical 值 —— 平行三元組、不是鏡像命名**：

| 側 | 起點 | 處理 | 結果 |
|---|-----|-----|-----|
| **外旋**（敏捷語言）| `外旋-Context` | `外旋-Constraints` | `外旋-Entropy` |
| **內旋**（榮格語言）| `內旋-覺察` | `內旋-反思` | `內旋-洞見` |

**內旋側改名對照**（舊值 → 新 canonical）：
- 舊 `內掘-時間與感知` → **`內旋-覺察`**（時間感 / 語義密度 = 覺察的頻譜層）
- 舊 `內掘-關係` → **`內旋-反思`**（關係深層結構 = 反思柱的人際版）
- 舊 `內掘-狀態流動` → **大多 → `內旋-洞見`**（狀態流動的果實端）；**少數 → `內旋-反思`**（還在處理階段的材料如陰影 / 情結）
- 舊 `內掘-道的縱軸` → **非三柱，保留為縱軸標記**（道 / 德 / 自由 / 無為 / 守柔 屬縱軸，不擠進三柱）
- 舊 純 `Entropy` / 純 `Constraints` → 看 note 歸哪一側 → 補 canonical 命名（外旋側保留 Entropy/Constraints 前綴、內旋側改為洞見/反思）
- 舊 `外旋-敏捷` → **太寬，細化到三柱之一**（隕石開發 → 外旋-Constraints 的反面；守柔 → 內旋-反思 / 道縱軸）
- 舊 `內旋-深度` → **`內旋-覺察`**（深度資料庫 = 覺察的來源側）

**術語統一**：內側統一用 `內旋-` 前綴（不再混用 `內掘-`）—— 與 [[內掘-心理]] MOC 的 `aliases: [內旋方法論, 內旋]` 對齊。

這輪先在 MOC 層把骨架牽起來；frontmatter 逐 note 改名交由 T7 P5 audit 執行。

**T7 P5 audit 完成（2026-04-15）**：35 個有 `pillars:` 的 notes 全部規範化為六個 canonical 值（零舊值殘留）。其餘 ~40 個 notes 留 `pillars: []` 是有意的 —— 標記為元層 / 底層 / 縱軸節點（[[網]] / [[共振]] / [[基頻]] / [[潛意識]] / [[集體無意識]] / [[道]] / [[德]] / [[自由]] 等），不入三柱。
